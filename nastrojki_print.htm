Тип обновления

Выбор типа обновления игрового клиента

HTTP

Обновление через HTTP сервер. Обновление клиента будет происходить скачиванием файлов которые должны быть размещены на НТТР сервере. НТТР сервер может стоять на другом удаленном ПК, главное чтобы файлы которые вы планируете обновлять были и на удаленном ПК. Данный тип обновления несколько не стабилен, медленный и ресурсоемкий.

Sockets

Прямое обновление. Launcher закачивает файлы напрямую через UpdSrv. Игровой клиент должен располагаться на том же ПК на котором установлен и UpdSrv. Данный тип обновления работает гораздо быстрее и более стабильный.

FTP

Обновление через FTP сервер. Идентично до первого метода. Только вместо протокола НТТР будет использоваться протокол FTP, а вместо веб-сервера будет использоваться FTP сервер.

Система архивации

вкл\выкл использование системы архивации.

Сохранение текущих параметров

После настройки всех параметров их необходимо сохранить нажав на данную кнопку.

Флажок автоматического запуска

Если отмечен то сервер автоматически начинает работу при запуске.

Остановка\запуск сервера

Останавливает или запускает работу сервера.

Включение\отключение античита

Данная опция включает дополнительную защиту на стороне Launcher’а. Для работы нужны дополнительные библиотеки. Так же данное поле активирует Anti-Cheat систему , если такая имеется.

Степень сжатия архивов

Степень сжатия файлов при использовании системы архивации. Чем лучше сжимает файлы тем больше понадобится времени на создание архива и сильнее будет нагрузка на ЦПУ.

Размер архива

Минимальный размер запрашиваемого обновления (в байтах) при котором будет создан архив.
 

Количество файлов для архива

Минимальное количество запрашиваемых файлов при котором будет сделан архив.

Мютекс сервера

Некоторые сервера при работе создают т.н. Мютексы, если сервер установлен на том же ПК что и UpdSrv, введите имя Мютекса для упрощения проверки статуса сервера.

Адрес сервера (статистика)

В данном поле вы можете указать IP-адрес Вашего сервера для того чтобы Launcher смог выводить статистику находится ли сервер в сети , или же он на данный момент вне сети.

Адрес сервера (запуск)

Данное поле используется только для игры Mu Online. Если заполнить данное поле то Launcher соединит main.exe именно с тем адресом сервера который Вы указали в данном поле.

Отображение статуса сервера

вкл\выкл проверку сервера (online\offline). Если данное поле не отмечено то вне зависимости от установленных данных в полях Server IP , Server Port (statistic) проверка статуса сервера производится не будет.

Порт сервера (статистика)

В данном поле вы можете указать порт Вашего сервера для того чтобы Launcher смог выводить статистику находится ли сервер в сети , или же он на данный момент вне сети.

Порт сервера (запуск)

Данное поле используется только для игры Mu Online. Если заполнить данное поле то Launcher соединит main.exe именно с тем портом сервера который Вы указали в данном поле.

Дополнительные параметры при запуске

в данном поле необходимо указывать дополнительные параметры для загрузки игры (Game File). Например: /u127.0.0.1 /p44405 (для игры MuOnline).

Файл запуска игры

Указывает относительный путь к файлу для запуска игры. Например если игровой файл находится в том же каталоге что и Launcher, то в данное поле необходимо вписать game.exe(Вам же, соответственно, нужно указать главный файл Вашей игры).

Добавление нового фильтра

Данный фильтр добавляет ключевые слова для фильтрации при чистке клиента. Например, если вы хотите чтобы Launcher не удалял скриншоты которые сохранены в папке с игрой необходимо добавить ключевое имя файла в список. Например в поле Name ввести Screenshot и нажать на кнопку Add. После этого будет добавлен новый фильтр. Фильтр работает по принципу поиску файлов в ОС Windows, можно использовать символы «*» , «?» и другие. Внимание! Если вы ввели Screenshot, то фильтр будет распространятся не только на точное имя файла , но и на косвенное! То есть Screenshot1.jpg, Screenshot2222.jpg ScreenshotByVasiaPypkin.exe так же будут подлежать фильтрации (фильтруемые файлы не будут удалены при чистке игрового клиента).

Поле добавления нового фильтра

Это поле используется только если установлен Clean game client when updating.

Список фильтров

В данном списке содержатся фильтры с помощью которых будет проходить чистка игрового клиента. Чтобы удалить нужный фильтр достаточно его выделить и нажать на кнопку Del на клавиатуре. Это поле используется только если установлен Clean game client when updating.

Пароль от FTP сервера для подключения. Это поле используется только если стоит тип обновления FTP.

Логин от FTP сервера

Логин от FTP сервера для подключения. Это поле используется только если стоит тип обновления FTP.

Адрес FTP сервера

Веб-адрес FTP сервера для подключения. Это поле используется только если стоит тип обновления FTP.

Порт сервера
